# Agentic RAG-VLM 优化与研究方向建议

这份文档基于原始项目架构（IROS 2026），为后续在 VLM 推理优化和 Agent 架构优化的 Robot-Agent 研究提供了具体的优化点与建议。

## 1. 当前项目分析与核心贡献总结

**核心贡献：**

* **HAA-RAG（分层物理可供性 RAG）：** 摒弃纯视觉检索，采用类别、物理可供性（类型/材质/易碎度/抓取区域）和视觉特征三级过滤检索抓取经验。
* **场景图约束推理（Scene Graph Reasoner）：** 用 VLM 理解空间关系（如支撑、遮挡、靠近易碎品），转化为具象的动作调整参数（如降低抓取力、侧向避障）。
* **Agentic 自反思规划（ReAct Loop）：** 具有基于 14 种错误分类的 3 级重试机制（调参->换抓取类型->重新规划），并引入了用于跨任务的短期情景记忆。

## 2. 核心优化方向一：记忆、检索与上下文管理系统

**目标：** 构建具备终身进化能力的物理可供性记忆库，解决复杂场景理解泛化与长时任务的“失忆”问题。

1. **基于 Graph RAG 的复杂场景空间关联检索**

   * **痛点：** 现有的 HAA-RAG 是以单物体为中心（Target-centric），缺乏对极度拥挤/遮挡场景结构特征的表征能力。
   * **具体方案：** 将目标周围的局部场景映射为多模态图结构 Embedding（融合 3D 先验或点云，包含物体节点和支撑/遮挡关联边）。遇到复杂堆叠场景时，不仅匹配单物体的抓取经验，更匹配数据库中具有类似“子图拓扑结构”的被遮挡/支撑抓取策略。
2. **结构化知识表示与终身学习 (Memory Consolidation / PlugMem)**

   * **痛点：** 目前的 116 条数据为静态死板的经验，且 14 种失败分类法为硬编码字典。
   * **具体方案：**
     * **经验提炼：** 在重试成功后，不只是记录 Log，而是将原始历程自动化提炼为结构化的**命题式知识**（环境事实）与**处方式知识**（抓取策略），动态追加到 ChromaDB。
     * **冲突与衰减：** 引入记忆的权重管理（LRU/衰减机制）。当特定静态知识频繁触发物理负反馈（Negative Rewards）时，自动削减其检索权重或利用 VLM 合并冲突条目，使其越用越聪明。
     * **开放词汇反思：** 废弃 14 种固定失败字典，在失败后通过 VLM 自由比对抓取前后的图像和力反馈特征，自主归因并生成下一次尝试的指令。
3. **上下文管理与状态机映射 (Context Caching & State Modeling)**

   * **痛点：** 长序列任务（如清理连续桌面的 3 个物品）中，历史图像和反馈堆积会导致严重的 Token 开销及指令淹没。
   * **具体方案：**
     * **渐进式上下文压缩：** 引入分层缓存与逐渐压缩机制（如 Micro Compact）。对短时动作执行序列，逐步清理图像帧和细碎日志，仅保留“工具调用骨架”和状态结论。针对 VLM 视觉历史，部署 SnapKV 或动态舍弃冗余帧。
     * **工作流状态化建模：** 基于 LangGraph 构建有向图（DAG）流程，将具身 Agent 从自由流浪模式转变为“节点-状态”显式追踪，从而随时可暂停、溯源或回滚。

## 3. 核心优化方向二：多 Agent 协作系统与工具调用增强

**目标：** 实现智能体角色的专业分工，赋予各 Agent 物理工具调用的健壮边界控制，构建高可信的"执行-评估-优化"闭环。

1. **具备专业分工的多具身 Agent 协作 (Multi-Agent Architecture)**

   * **方案设计 (借鉴 MUSE / OpenClaw 架构)：**
     * `Architect/Vision Agent`：负责复杂场景的第一步解析，构建 Scene Graph，计算物理可供性空间。
     * `Planning-Execution Agent`：提取意图并专注于抓取和机械臂的规划执行，专注于调用“物理移动、力控”等低维控制工具。
     * `Reflect/Critic Agent`：作为监督者，针对 Execution Agent 返回的执行结果和终端传感器数据，负责进行成功/失败校验、开放词汇归因，以及整理新的知识沉淀。
2. **工具调用的最小集与闭环验证 (Robust Tool Use)**

   * **具体方案：**
     * **工具组拆分：** 采用“最小可用工具集设计”，将具身交互能力封装为原子级实体操作（如 `move_to_pose`, `adjust_grip_force`, `query_memory`, `extract_scene_graph`）。
     * **执行异常与回退 (Error+Fallback)：** 在底层包装工具运行函数，加入异常拦截 `Tool Result Handler`。即使工具（IK 求解失败或抓取中途脱落）执行崩溃，也不导致主流程崩溃，而是以结构化的错误信息返回给 Agent，让其激活元认知，自主决定是继续重试、寻找新工具，还是上报求助（Human-in-the-loop）。
     * **可审计型执行日志：** 每个 Tool-call 动作均需附带一个局部的“决策理由”（Why did I do this?），方便后续调试及反思模型的快速定位分析。

## 4. 核心优化方向三：VLM 大脑与动作模型 (Action Model) 小脑集成

**目标：** 将纯基于视觉坐标与硬编码逆运动学 (IK) 的控制架构升级为“高层规划+轻量操作”混合范式 (Hybrid Architecture)，进一步提高操作的连续性、拟人度及容错性。

1. **阶段一：VLM 视觉定位 (Grounding) 与 运动学 (IK) 的闭环打通**
   * **痛点：** 以前的坐标通过硬编码获取 `storage_bin` 相关，或直接调用仿真器底层的作弊数据。
   * **具体方案：** 
     * 本地部署 `models/Qwen3-VL-8B`。使用 ManiSkill 渲染的 RGB(D) 图像直接要求 VLM 圈定目标的 2D BBox 并辅助计算 `(x, y, z)`，作为真实的 Task Planner 取代人造点位输入。
     * 由 LAMA 现有的 `node_execution_agent` (基于 `pose_utils.py`) 承接解析出的坐标并进行执行测试。

2. **阶段二：从硬编码 (IK) 到数据驱动 (ACT/SmolVLA) 的动作生成**
   * **痛点：** IK 几何路径（直线插补）过于生硬，在复杂场景易碰撞且缺乏自适应容错。
   * **具体方案：** 
     * 在验证了步骤一中 VLM Plan 的可靠性后，将 LAMA-VLM 中的 `Execution Agent` 底层剥离。
     * 动作控制交由 ACT 等基于模仿学习的轻量级动作模型 (工作频率 ~50Hz)。这些策略模型根据当前的多模态输入、本地感受滚动输出连续序列 (Action Chunks)。

3. **硬件级部署验证规划 (RTX 5090)**
   * 利用拥有极高访存带宽与 32GB 显存的 RTX 5090 单卡机器。
   * VLM 大脑推荐使用 `Qwen3-VL-8B` (或者同系列的 `Qwen3.5-VL-8B/14B`) BF16 或 INT4 精度，占用显存 < 18GB。该系列原生支持卓越的视觉定位 (Bounding Box / 3D Points)。
   * 剩余显存能够绰绰有余地同时运行 50Hz 闭环的轻量动作模型 (ACT ~1-2GB) 以及 ManiSkill / Isaac Sim 等强化学习物理仿真引擎。

## 5. 结论与优先级建议

通过引入**Graph RAG与知识长程衰减机制**，可以最直接地扩容现有项目的“记忆库”边界；同时通过架构层引入 **Critic Agent 监督与状态压缩机制**，能有力控制长时间具身执行的乱步与幻觉。这个大优化方案能有效让 `Agentic-RAG-VLM` 系统摆脱现有“规则硬编码框架”和“上下文爆炸瓶颈”，走向真实具身环境下的架构级前沿！

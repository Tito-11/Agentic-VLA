# LAMA-VLM 组会汇报稿

## 1. 项目定位与目标

本项目聚焦机器人在复杂桌面场景中的连续抓取任务，核心目标是把“看得懂”与“做得对”统一起来：

- 看得懂：通过视觉语言模型理解目标物体与空间关系
- 做得对：在物理执行中具备异常检测、失败归因、参数修正与再尝试能力
- 能进化：将成功与失败经验写入长期记忆，使策略随任务不断优化

我们将原有 Agentic RAG-VLM 体系升级为 LAMA-VLM（Lifelong Autonomous Multi-Agent VLM for Robotic Manipulation），形成了可解释、可回溯、可重试的完整闭环。

---

## 2. 系统框架（详细版，可直接讲解）
系统采用五层闭环架构：感知层、检索记忆层、决策编排层、执行层、反思层。
LAMA-VLM 采用五层闭环架构：感知层、检索记忆层、决策编排层、执行层、反思层。核心数据流如下：

Environment Observation -> Vision Agent -> Graph RAG / Memory Retrieval -> Planner -> Execution Tools -> Critic -> (Success -> Transport) / (Failure -> Re-plan) -> Memory Consolidation

可以把它理解为：

- VLM 负责“看懂场景”
- RAG 和 Memory 负责“想起该怎么做”
- StateGraph 负责“按规则推进流程”
- Toolchain 负责“稳健执行动作”
- Critic 负责“验收与纠错”
- Memory Consolidation 负责“把本次经验变成下一次能力”

这一架构对应你的创新目标：不仅要抓起来，还要抓得稳、失败可恢复、经验可进化。

### 2.1 感知层：VLM 视觉理解与定位

输入为仿真环境图像（RGB/RGBD），由本地 Qwen3-VL-8B 负责生成动作可用的视觉语义信息：

- 目标物体定位（Grounding）
- 场景关系理解（邻近、遮挡、可接近性）
- 姿态先验提示（例如 target_yaw，避免“先碰撞再重试”）

这一层的核心创新是把 VLM 从“坐标回归器”升级为“执行前决策前端”：

- 在动作发生前就给出风险信息（邻近障碍、可接近方向）
- 在策略层提前注入姿态先验，而不是碰撞后被动修正

展示价值：先看懂再行动，减少无效动作和首抓碰撞率。

### 2.2 检索与记忆层：Graph RAG + Lifelong Memory（Evo-KAM）

传统 target-centric 检索只看目标本体，LAMA-VLM 改为“目标 + 邻居 + 关系”的结构化检索：

- Topo-Graph RAG：利用 supports / occluded_by / next_to 构建局部拓扑子图，检索相似场景策略
- Lifelong Memory：把成功与失败轨迹提炼成可复用先验（z_offset、force、姿态）
- 记忆权重更新：出现冲突或高失败率时，动态衰减低质量经验权重

这一层体现两项关键创新：

- 结构感知检索创新：从“目标中心检索”升级为“拓扑子图检索”，在拥挤和遮挡场景下更稳
- 终身记忆进化创新：把失败转化为下一次策略修正，让系统具备“越用越准”的能力

展示价值：从“查固定规则”升级为“查经验 + 修经验 + 强化经验”。

### 2.3 决策编排层：DAG 状态机（LangGraph）

核心状态流为：

Vision -> Planning -> Execution -> Critic -> Transport

其中 Critic 是条件路由核心：

- 成功路径：Critic = pass，进入 Transport 放置
- 失败路径：Critic = fail，回到 Planning 做参数修正与重试
- 边界路径：重试次数耗尽，跳过当前目标并继续任务

与“脚本串行调用”相比，这一层的创新是显式状态化：

- 节点可追踪（哪一步失败一目了然）
- 路由可解释（为什么重试/为什么放弃有证据）
- 流程可回溯（方便调试、消融和答辩）

这一层对应创新点：DAG-EAM（Directed Acyclic Graph Explicit Action Modeling）。

展示价值：把多智能体从“概念协作”变成“可验证协作”。

### 2.3.1 多Agent系统如何体现（角色分工 + 交互时序）

为避免“看起来只是一个大脚本”，这里把多Agent边界明确成 4 个专业角色。

角色一：Architect/Vision Agent

- 输入：当前观测图像、目标对象名
- 输出：目标坐标、场景关系、候选姿态先验
- 职责：负责“看懂场景与目标状态”，只做感知与语义解析，不直接下发低层动作

角色二：Planning Agent

- 输入：Vision 输出 + Graph RAG 检索结果 + Lifelong Memory 先验
- 输出：可执行参数包（如 z_offset、force、target_yaw、重试预算）
- 职责：负责“决策如何抓”，把语义信息转换成可执行策略

角色三：Execution Agent

- 输入：Planning 参数包
- 输出：工具调用日志与执行结果
- 职责：负责“真正执行动作”，通过 robust_move / robust_grasp / robust_release 驱动机器人

角色四：Reflect/Critic Agent

- 输入：Execution 日志 + 终端状态反馈（如提拉后目标高度变化）
- 输出：通过/失败判定 + 失败归因 + 回路决策
- 职责：负责“验收和纠错”，决定进入 Transport 还是返回 Planning 重试

这一分工不是概念堆叠，而是有明确数据契约：

- Vision 只负责感知语义，不直接控制机械臂
- Planning 只负责产出策略，不直接执行工具
- Execution 只负责动作落地，不定义成功标准
- Critic 独立掌握验收权，避免“自己执行自己判成功”的偏差

一次任务中的时序交互如下：

步骤1：Vision Agent 解析当前目标和邻域关系

步骤2：Planning Agent 检索记忆并生成执行参数

步骤3：Execution Agent 调用工具完成接近、夹取、提拉

步骤4：Critic Agent 判断是否真实抓取成功

步骤5：若成功则进入 Transport 放置，若失败则触发重定位与参数修正后回到步骤2

这部分就是你“多Agent系统”的最直接体现：角色专业化、职责解耦、状态可追踪、失败可回路。

### 2.4 执行层：Audit-Driven Robust Toolchain

所有物理动作被封装为原子工具：

- robust_move
- robust_grasp
- robust_release

并引入两类工程保障：

- 异常拦截与回退（Error + Fallback），避免 IK/碰撞/滑脱直接打崩主流程
- 审计式日志（每个工具调用附带 reason），支持 Critic 做可追责归因

这里对应你的创新点D：审计驱动的鲁棒执行链。

关键价值是把“动作能跑”升级成“动作可诊断、可复现、可优化”。

### 2.5 反思闭环层：Critic + Retry + Re-localization

系统通过抓取后验证（例如提拉后的目标高度变化）判断“是否真的抓住”：

- 若成功：进入放置流程并回写正向经验
- 若失败：触发错误归因、动态重定位、参数修正（z_offset、force、姿态）并重试

这一层融合了两项落地创新：

- 视觉先验防碰撞：执行前注入姿态先验，降低邻近障碍干涉
- 实时观测闭环修复：用实时状态替代静态快照，避免“抓起却被误判失败”

展示价值：真正做到“失败才重试，成功就放行”。

### 2.6 一句话总结系统框架

LAMA-VLM 的系统框架本质上是一个“可解释的具身闭环控制器”：

- 前端靠 VLM 和 Graph RAG 看懂场景
- 中间靠 StateGraph 和 Toolchain 稳定执行
- 后端靠 Critic 和 Memory 持续纠错与进化

因此它不仅完成任务，还能在任务中学习并逐步提升成功率与稳定性。

## 4. 实验结果与阶段结论

### 4.1 当前公开基准结果（README）

- Single Grasp: 91.7%
- Interactive Task: 64.2%
- Long-Horizon: 66.7%
- Overall: 78.3%

### 4.2 结果解读（组会可直接说）

- SG 明显高于 IT/LH，说明单步抓取能力已较稳定
- IT/LH 下降，说明多步链路中的误差累积仍是主要瓶颈
- 这与当前工作重点一致：通过 Critic 回路、记忆修正、姿态先验来抑制长时漂移

### 4.3 阶段结论

- 框架已从“可运行”发展为“可反思、可回溯、可演进”
- 已具备复杂场景下的端到端执行闭环
- 下一步重点为实验设计、消融对比与统计显著性验证

### 4.4 已验证的关键修复价值

- 修复静态状态读取导致的误判重试后，成功抓取可正确进入放置流程
- 引入防干涉姿态先验后，邻近障碍场景的无效碰撞重试显著减少
- 执行层差分姿态控制替代粗暴角度写入后，解算稳定性明显提升

## 5. 下一步实验计划（可用于答辩）

1. 消融实验：
- 去掉 Graph RAG
- 去掉 Critic
- 去掉 Lifelong Memory
- 去掉姿态先验
比较成功率、重试次数与任务时延。

2. 稳定性实验：
- 多随机种子
- 多物体摆放密度
- 遮挡强度分级
统计方差与失败类型分布。

3. 长时任务实验：
- 连续任务长度扩展
- 上下文压缩前后对比
评估 Token 开销、决策延迟与成功率变化。
